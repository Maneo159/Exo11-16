================================================================================
                            NOTES IMPORTANTES
================================================================================

1. L'API est réactive (non-bloquante) - Les réponses utilisent Mono/Flux
2. Tous les timestamps (created_at, updated_at) sont gérés automatiquement
3. L'email doit être unique (contrainte UNIQUE en base de données)
4. Les codes HTTP utilisés:
   - 200 OK: Requête réussie (GET, PUT, POST)
   - 201 Created: Ressource créée (POST)
   - 204 No Content: Suppression réussie (DELETE)
   - 404 Not Found: Ressource non trouvée
   - 400/409: Erreur de validation (email en doublon, etc.)

5. Format des réponses:
   - GET multiple: Array JSON
   - GET simple: Object JSON
   - POST: Object JSON créé
   - PUT: Object JSON mis à jour
   - DELETE: 204 No Content (pas de corps)

6. Pour réinitialiser la base de données:
   - Arrêter l'application
   - Arrêter Docker Compose: docker-compose down
   - Nettoyer les volumes: docker volume rm exo11_postgres_data
   - Relancer: docker-compose up -d
   - Redémarrer l'application
